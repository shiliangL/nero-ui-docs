RNStatusAction 状态操作组件 ​

组件介绍 ​

RNStatusAction 是一个状态操作组件，用于展示和处理数据状态的变更，支持多种状态类型和自定义操作。基于 Naive UI 的组件封装。

Props ​

属性名类型默认值说明
statusstring/number-当前状态值
statusMapObject{}状态映射配置对象
rowObject{}当前行数据
sizestring'small'标签尺寸，可选值：'small'/'medium'/'large'
disabledbooleanfalse是否禁用操作
placementstring'bottom'下拉菜单位置

状态映射配置 ​

statusMap对象应包含以下结构：

typescript
interface StatusItem {
  label: string; // 状态显示文本
  value: string | number; // 状态值
  type?: string; // 状态类型，用于显示不同样式
  color?: string; // 自定义颜色
  actions?: Array<{
    // 可进行的操作列表
    label: string; // 操作文本
    value: string | number; // 目标状态值
    type?: string; // 操作按钮类型
    permission?: string; // 权限标识
    confirm?: {
      // 确认框配置
      title: string;
      content: string;
    };
  }>;
}

type StatusMap = Record<string | number, StatusItem>;

事件说明 ​

事件名说明回调参数
change状态变更时触发(newStatus: string/number, oldStatus: string/number, row: Object)
action-click操作按钮点击时触发(action: Object, status: string/number, row: Object)

使用示例 ​

基础用法 ​

vue
<template>
  <rn-status-action
    :status="currentStatus"
    :status-map="statusMap"
    :row="rowData"
    @change="handleStatusChange"
  />
</template>

<script setup>
import { ref } from 'vue';

const currentStatus = ref('pending');
const rowData = ref({
  id: 1,
  name: '订单1',
});

const statusMap = {
  pending: {
    label: '待处理',
    value: 'pending',
    type: 'warning',
    actions: [
      {
        label: '通过',
        value: 'approved',
        type: 'success',
        confirm: {
          title: '确认通过',
          content: '是否确认通过该申请？',
        },
      },
      {
        label: '拒绝',
        value: 'rejected',
        type: 'error',
      },
    ],
  },
  approved: {
    label: '已通过',
    value: 'approved',
    type: 'success',
  },
  rejected: {
    label: '已拒绝',
    value: 'rejected',
    type: 'error',
  },
};

const handleStatusChange = (newStatus, oldStatus, row) => {
  console.log('Status changed:', { newStatus, oldStatus, row });
};
</script>

带权限控制 ​

vue
<template>
  <rn-status-action
    :status="status"
    :status-map="statusMapWithPermission"
    :row="rowData"
  />
</template>

<script setup>
const statusMapWithPermission = {
  pending: {
    label: '待审核',
    value: 'pending',
    type: 'warning',
    actions: [
      {
        label: '审核通过',
        value: 'approved',
        type: 'success',
        permission: 'audit:approve',
      },
      {
        label: '审核拒绝',
        value: 'rejected',
        type: 'error',
        permission: 'audit:reject',
      },
    ],
  },
  // ... 其他状态配置
};
</script>

自定义样式 ​

vue
<template>
  <rn-status-action
    :status="status"
    :status-map="customStatusMap"
    :row="rowData"
  />
</template>

<script setup>
const customStatusMap = {
  processing: {
    label: '处理中',
    value: 'processing',
    color: '#1890ff',
    actions: [
      {
        label: '完成',
        value: 'completed',
        type: 'primary',
      },
    ],
  },
  completed: {
    label: '已完成',
    value: 'completed',
    color: '#52c41a',
  },
};
</script>

注意事项 ​

  1. statusMap中的每个状态必须有唯一的value值
  2. 操作按钮的权限控制需要与全局权限系统配合
  3. 确认框配置可以避免误操作
  4. 状态变更时注意处理相关的业务逻辑
  5. 可以通过type或color自定义状态标签的样式
