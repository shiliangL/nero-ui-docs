Vue.js 前端代码审核指南 📋 ​

使用说明本指南提供了全面的 Vue.js 代码审核标准，帮助团队保持代码质量和一致性。请在每次代码审核时参考此指南。:::

vue

1. 项目结构审核 🏗️ ​

1.1 目录结构规范 ​

目录结构推荐的 Vue.js 项目目录结构：

src/
├── assets/        # 静态资源
├── components/    # 通用组件
├── views/         # 页面组件
├── router/        # 路由配置
├── store/         # 状态管理
├── utils/         # 工具函数
└── types/         # TypeScript 类型定义
  • 确保项目遵循 Vue.js 推荐的目录结构
  • 检查是否合理分离 components、views、router、store、utils 等目录
  • 验证静态资源（图片、样式、字体等）是否合理组织

1.2 命名规范 ​

命名约定统一的命名规范有助于提高代码可读性和维护性 :::

类型规范示例
组件文件PascalCaseUserProfile.vue
路由页面以 Page/View 结尾UserProfilePage.vue
工具函数camelCaseformatDate.js
CSS 类名kebab-caseuser-avatar

2. 组件开发规范 🔧 ​

2.1 组件基础规范 ​

组件开发最佳实践遵循 Vue3 组合式 API 的最佳实践，确保组件的可维护性和可复用性 :::

vue
<script setup lang="ts">
// ✅ 推荐的组件结构
interface Props {
  title: string;
  items?: string[];
}

const props = defineProps<Props>();
const emit = defineEmits<{
  (e: 'update', value: string): void;
}>();
</script>
  • 组件名应该是多个单词（避免与 HTML 元素冲突）
  • Props 定义应该尽可能详细，包含类型和默认值
  • 使用 defineProps、defineEmits 等 Vue3 组合式 API
  • 确保组件单一职责原则

2.2 组件通信 ​

组件通信方式根据场景选择合适的通信方式：

  • Props/Emits：父子组件通信
  • Provide/Inject：跨层级组件通信
  • Pinia：全局状态管理 :::
vue
<!-- ✅ 推荐的 Props 定义方式 -->
<script setup lang="ts">
const props = defineProps<{
  modelValue: string;
  label?: string;
  required?: boolean;
}>();

const emit = defineEmits<{
  (e: 'update:modelValue', value: string): void;
  (e: 'change', value: string): void;
}>();
</script>

3.2 TypeScript 规范 ​

TypeScript 最佳实践

  • 优先使用 interface 定义对象类型
  • 避免使用 any，使用 unknown 代替
  • 合理使用泛型增加代码复用性 :::
typescript
// ✅ 推荐的类型定义方式
interface IUserProfile {
  id: number;
  name: string;
  email: string;
  preferences?: UserPreferences;
}

type UserPreferences = {
  theme: 'light' | 'dark';
  notifications: boolean;
};

3.3 业务逻辑 ​

注意事项

  • 保持业务逻辑清晰简单
  • 做好错误处理和边界情况
  • 避免代码重复 :::
typescript
// ✅ 推荐的错误处理方式
const fetchUserData = async (userId: string) => {
  try {
    const response = await api.get(`/users/${userId}`);
    return response.data;
  } catch (error) {
    console.error('Failed to fetch user data:', error);
    throw new Error('用户数据获取失败');
  }
};
